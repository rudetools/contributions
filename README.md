# RudeTools – Proposer un outil

Ce dépôt est dédié aux **demandes d’ajout d’outils** pour le [référentiel RudeTools](https://rudetools.dev).

## Comment proposer un outil ?

### Option 1 : Ouvrir une issue (recommandé)

Ouvrez une [nouvelle issue](https://github.com/rudetools/contributions/issues/new) en indiquant :

- **Nom de l’outil**
- **URL** (site officiel ou dépôt GitHub)
- **Catégorie** (ex. Containers & Kubernetes, Monitoring, CI/CD, Sécurité… — voir [rudetools.dev](https://rudetools.dev) pour la liste)
- **Courte description** (à quoi sert l’outil, pour qui)
- **Licence** (si connue, ex. MIT, Apache-2.0)

Votre suggestion sera examinée et pourra être intégrée au référentiel.

### Option 2 : Ouvrir une Pull Request

Si vous souhaitez proposer une **fiche rédigée** (format utilisé sur le site), vous pouvez :

1. Forker ce dépôt
2. Ajouter un fichier décrivant l’outil (par ex. dans un dossier `suggestions/` ou à la racine)
3. Ouvrir une Pull Request

Indiquez la catégorie cible et, si possible, respectez la structure des fiches du site (voir le [guide de contribution](https://rudetools.dev/contribuer) sur RudeTools).

---

Les outils proposés doivent être **open source** et pertinents pour **DevOps, SRE, CI/CD, infrastructure, sécurité ou IT** en général.

Merci pour votre contribution.
